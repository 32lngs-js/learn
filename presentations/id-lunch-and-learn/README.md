# What One Engineer Learned About Instructional Design

A 10-minute presentation for a lunch-and-learn with ~20 instructional designers.

## How to present

Open `index.html` in any modern browser. For the best experience, press **F** to enter fullscreen once the page loads.

## Controls

| Key | Action |
|-----|--------|
| Arrow Right / Space / Arrow Down | Next slide |
| Arrow Left / Arrow Up | Previous slide |
| N or S | Toggle speaker notes |
| O or Esc | Toggle overview grid |
| F | Toggle fullscreen |
| Home | Jump to first slide |
| End | Jump to last slide |

You can also click the left/right halves of the screen to navigate, or swipe on touch devices.

## Deep linking

Each slide is addressable via URL hash: `index.html#/3` goes directly to slide 3.

## Speaker notes

Press **N** to reveal the speaker notes panel at the bottom of the screen. Notes include pacing reminders for key moments.

## Slide 7: demo video

Slide 7 plays `assets/demo.mp4` — a ~90s Palestra walkthrough built in Remotion. The video plays from the start each time you enter the slide and pauses on exit. If you need to rebuild it, see `remotion/README.md`.

## Structure (10 slides)

1. Title / Opening
2. The Scale (10 courses, real curriculum cards)
3. The Problem (AI without design)
4. The Process (6-step software planning path with feedback loops)
5. The Analogies (my path ↔ what IDs already do)
6. What AI is good/bad at (the real mechanism behind the path)
7. Demo (Palestra predict → calibration → interview)
8. Evidence (pedagogical features)
9. The Landscape (value of design expertise)
10. Close / Call to collaborate + repo invitation
