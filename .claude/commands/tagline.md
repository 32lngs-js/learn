# Tagline & Messaging Generator

Create compelling taglines and messaging that align with the brand positioning.

## Inputs

- **Brand/product name** (if decided)
- **Positioning statement** (if available — run /positioning first if not)
- **Target audience** for this tagline (general, career switchers, students, professionals, etc.)
- **Tone** (inspiring, playful, authoritative, warm, bold)
- **Context** where the tagline will appear (homepage hero, ad campaign, app store, social bio, email subject)

If the user doesn't provide these, ask before proceeding.

## Process

### Phase 1 — Research Competitors

Search for taglines used by:
- Duolingo, Brilliant, Coursera, Khan Academy, Codecademy, Skillshare, Udemy, DataCamp
Note what works, what's overused, and where gaps exist.

### Phase 2 — Generate Variants (20+)

Generate at least 20 tagline candidates across these styles:
1. **Promise-based** — what the user gets ("Master anything, one spark at a time")
2. **Challenge-based** — dare the user ("Think you know poker math?")
3. **Identity-based** — who the user becomes ("Built for builders")
4. **Action-based** — imperative verb ("Learn by doing. Prove by testing.")
5. **Metaphor-based** — evocative imagery ("Your brain's gym membership")
6. **Minimal** — ultra-short, punchy (3-5 words max)

### Phase 3 — Evaluate & Rank

For each candidate, assess:
- **Clarity:** Does someone immediately understand the product?
- **Distinctiveness:** Could a competitor use this? If yes, it's too generic.
- **Emotional pull:** Does it make someone want to try the product?
- **Versatility:** Does it work across hero banner, social bio, and ad copy?

### Phase 4 — Messaging Pyramid

For the top 5 taglines, build out:
- **Primary tagline** (the one-liner)
- **Supporting statement** (1-2 sentences expanding on the promise)
- **Proof points** (3 bullets — e.g., "AI-powered practice interviews", "Spaced repetition that sticks", "Earn Sparks as you master skills")

### Phase 5 — Audience Variants

Adapt the top 2 taglines for specific audiences:
- Career switchers / job seekers
- Students / exam prep (CFA, technical interviews)
- Developers (React, system design, Claude Code)
- Casual learners (poker, general curiosity)

Present final recommendations with context for where each works best.
