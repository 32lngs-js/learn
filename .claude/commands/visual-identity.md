# Visual Identity Brief

Design a complete visual identity system for the platform.

## Inputs

- **Brand name and tagline** (run /brand-name and /tagline first if needed)
- **Brand archetype(s)** (e.g., Sage + Creator, Explorer, etc.)
- **Audience** (developers, students, professionals, casual learners)
- **Existing visual references** the user likes (URLs, screenshots, competitor examples)
- **Constraints** (must work in dark mode, must be accessible, existing colors to keep)

If the user doesn't provide these, ask before proceeding.

## Process

### Phase 1 — Mood & Direction

Research visual identities of:
- Duolingo (playful, mascot-driven, green)
- Brilliant (clean, mathematical, orange/dark)
- Linear (minimal, professional, purple gradients)
- Notion (warm, approachable, black/white/beige)
- Stripe (bold, gradient-heavy, professional)

Present 3 possible visual directions with rationale:
1. **Playful & Energetic** — bright colors, rounded shapes, illustrations
2. **Clean & Professional** — muted palette, sharp type, whitespace-forward
3. **Bold & Technical** — dark backgrounds, accent colors, code-inspired

Ask the user to pick a direction or blend.

### Phase 2 — Color System

Define:
- **Primary color** (brand color — used for CTAs, logo, key accents)
- **Secondary color** (complementary — used for secondary actions)
- **Semantic colors** (success/green, warning/amber, error/red, info/blue)
- **Neutral scale** (background, surface, border, text — 8-10 steps)
- **Dark mode variants** for each

Provide hex values, HSL values, and contrast ratios (WCAG AA compliance).

### Phase 3 — Typography

Recommend:
- **Heading font** (display/impact)
- **Body font** (readable at small sizes)
- **Monospace font** (for code blocks)
- Font pairing rationale
- Scale system (text-xs through text-4xl with rem values)

Prefer Google Fonts or open-source fonts for web use.

### Phase 4 — Logo Direction

Provide 3 logo concept briefs (for a designer to execute):
- **Concept name** and description
- **Style** (wordmark, lettermark, icon + wordmark, abstract mark)
- **Key visual element** (what makes it memorable)
- **Color usage** (which brand colors)
- **Variations needed** (full, icon-only, monochrome, dark bg)

### Phase 5 — Illustration & Iconography

Define:
- **Illustration style** (flat, isometric, hand-drawn, 3D, none)
- **Icon style** (outlined, filled, duotone — reference Lucide/Heroicons)
- **Character/mascot** direction (if applicable)
- **Image treatment** (photography style, overlays, gradients)

### Phase 6 — Asset Checklist

List all assets that need to be created:
- Logo files (SVG, PNG at various sizes)
- Favicon and PWA icons
- OG image template
- Social media profile images and banners
- Email header template
- Presentation deck template
- App store screenshots (if applicable)

Present the full identity guide as a structured document the user can hand to a designer.
